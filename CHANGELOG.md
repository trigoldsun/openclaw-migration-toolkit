# Changelog

所有重要的变更都会记录在这个文件中。

## [2.0.0] - 2026-04-16

### 🎉 重大更新

这是 v2.0 版本，包含架构重构和多项功能改进。

### ✨ 新增功能

#### 1. OpenClaw 发现机制 (`discover` 命令)
- 自动发现 OpenClaw 安装位置和版本
- 检测 Gateway 配置和运行状态
- 列出所有 Provider 连接状态
- 统计 Skills 和 Memory 文件数量
- 发现 Cron 任务
- 支持 JSON 格式输出 (`--json`)
- 支持详细输出 (`--verbose`)

#### 2. 增量同步 (`sync` 命令)
- 支持 rsync 断点续传
- 带宽限制支持 (`--bw-limit`)
- 灵活的组件选择 (`--components`)
- 智能排除模式 (自动排除 session, auth, logs 等)
- 多次同步只传输变更部分

#### 3. 加密传输
- GPG 加密支持 (AES256)
- OpenSSL 加密备用方案
- 导出时加密 (`--encrypt --password`)
- 导入时解密 (`--decrypt --password`)
- 传输加密选项

#### 4. 交互式迁移向导 (`interactive` 命令)
- 6 步引导式迁移流程
- 组件选择向导
- 目标服务器配置
- 传输方式选择
- 加密选项
- 执行前确认

#### 5. Canvas 状态迁移
- 新增 Canvas 状态导出/导入支持
- 迁移完整的 Canvas UI 状态

### 🔧 架构改进

#### OpenClaw 架构正确理解
- **重要**: 重写时正确理解了 OpenClaw 架构
- Gateway 是控制平面，不是数据存储
- 配置通过 WS API 管理，不是简单文件复制
- Provider session 不能直接复制，需要重新认证
- 设备配对机制需要单独处理

#### 路径发现机制
- 自动搜索 OpenClaw 命令位置
- 支持多种安装路径:
  - `/usr/local/bin/openclaw`
  - `~/.local/bin/openclaw`
  - `/snap/bin/openclaw`
  - `/opt/openclaw/bin/openclaw`
- 环境变量配置支持

#### Provider 迁移改进
- 只迁移 Provider 元数据
- 自动跳过敏感 session 文件
- 生成 session_info 标记文件
- 明确提示需要重新认证

### 📊 组件迁移改进

| 组件 | v1.0 | v2.0 |
|------|------|------|
| Gateway Config | 简单复制 | 脱敏处理 + jq 解析 |
| Providers | 全部复制 | 仅元数据 + session 跳过 |
| Skills | ✓ | ✓ |
| Memory | ✓ | ✓ |
| Sessions | ✓ | 可选 |
| Cron | 文本导出 | JSON + 可执行恢复脚本 |
| Env | 简单脱敏 | 改进的脱敏逻辑 |
| Canvas | ✗ | ✓ 新增 |

### 🔒 安全改进

1. **敏感信息处理**
   - Gateway 配置中的 secret/token/key/password/auth 字段自动脱敏
   - 使用 jq 进行精确字段处理
   - Provider session 文件自动跳过

2. **加密备份**
   - GPG AES256 加密
   - OpenSSL AES-256-CBC 备用
   - PBKDF2 密钥派生

3. **传输安全**
   - SSH/SCP 加密传输
   - 可选端到端加密

### 📝 日志和改进

1. **日志系统增强**
   - 独立日志目录
   - 时间戳记录
   - 分类日志 (INFO/SUCCESS/WARN/ERROR/DEBUG)
   - Verbose 模式支持

2. **错误处理**
   - `set -euo pipefail` 严格模式
   - ERR trap 错误捕获
   - 详细的错误信息

3. **用户界面**
   - 彩色输出
   - 进度指示
   - 详细的成功/警告/错误提示

### 🐛 Bug 修复

1. 修复了 openclaw 命令路径假设错误
2. 修复了环境变量脱敏不完整问题
3. 修复了静默忽略错误的静默处理
4. 修复了 Cron 导出格式问题

### 📚 文档改进

1. 新增本 CHANGELOG.md

3. 迁移前/中/后检查清单

### 🔄 向后兼容性

## [1.0.0] - 2024-01-15

### 🎉 初始版本

初始发布的 OpenClaw Migration Toolkit。

### 功能

- `export` - 导出 OpenClaw 数据到备份文件
- `import` - 从备份文件导入数据
- `migrate` - 服务器间迁移
- `status` - 查看迁移状态
- `validate` - 验证备份文件完整性
- `cleanup` - 清理旧备份

### 支持的组件

- Gateway 配置
- 插件
- Skills
- Memory 文件
- Sessions (可选)
- Cron 任务 (文本格式)
- 环境变量 (简单脱敏)

### 限制 (v1.0 已知问题)

- ⚠️ OpenClaw 路径假设错误 (假设 `$HOME/openclaw`)
- ⚠️ 简单文件复制，不理解 OpenClaw 架构
- ⚠️ Provider session 直接复制 (不安全)
- ⚠️ 敏感信息处理不完善
- ⚠️ 无加密传输
- ⚠️ 无增量同步

## 迁移指南 (从 v1.0 升级)

### 命令兼容

v2.0 完全兼容 v1.0 的命令：

```bash
# v1.0 命令仍然有效
./migrate.sh export -o ~/backups
./migrate.sh import -i ~/backups/openclaw-backup-*.tar.gz
./migrate.sh migrate -t user@target-server -p 22
```

### 新增命令

```bash
# 发现本地 OpenClaw (新增)
./migrate.sh discover

# JSON 格式输出 (新增)
./migrate.sh discover --json

# 增量同步 (新增)
./migrate.sh sync -t user@server --incremental

# 交互式向导 (新增)
./migrate.sh interactive

# 加密导出 (改进)
./migrate.sh export --encrypt --password mysecret

# 指定组件 (改进)
./migrate.sh export -c config,skills,memory
```

### 重要变更

1. **备份文件名**: 格式不变，但 manifest.json 新增 `toolkit_version` 字段
2. **Provider 处理**: v2.0 不会复制 session 文件，导入后需要重新认证
3. **敏感字段**: v2.0 使用 jq 进行更精确的脱敏处理
